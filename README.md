# MOODLE LESSON BRUTEFORCER from futsurignnahateu

Программа для автоматического прохождения лекций в системе Moodle. Скрипт самостоятельно решает все вопросы, перебирая варианты ответов, и пролистывает содержимое лекции.

Брутфорсер был сделан для курсов "Право" и "Правовые основы профессиональной деятельности и защиты прав на объекты интеллектуальной собственности" для студентов ЛЭТИ, работоспособность брутфорсера на других курсах не гарантируется.
## Что BruteForcer делает?

- Автоматически решает вопросы с одним правильным ответом
- Перебирает варианты ответов до получения правильного
- Пролистывает страницы с текстом (контент-страницы)
- Работает в двух режимах: с чтением лекции и с автопролистыванием
- Ведет подробный лог всех действий

Приложение в разработке, позднее может быть усовершенствовано по мере необходимости.

## Режимы работы

### Режим 1: Чтение лекции (y)

```
Читать лекции (ждать вопросы)? (y/n): y
```

Скрипт будет:
- Ждать появления вопросов на странице
- Проверять наличие вопросов каждые 0.5 секунды
- Вывести сообщение "Вопросы не найдены" если их нет
- Вы можете пролистывать лекцию вручную в браузере

### Режим 2: Автопролистывание (n)

```
Читать лекции (ждать вопросы)? (y/n): n
```

Скрипт будет:
- Автоматически нажимать кнопку "Дальше" на контент-страницах
- Выводить точки "." при каждом клике
- Самостоятельно пролистывать всю лекцию
- Останавливаться только на вопросах

## DEBUG режим

DEBUG режим выводит дополнительную информацию о работе скрипта.

```
Нужны ли детальные DEBUG логи? (y/n): y
```

При включении видите:
- Номер каждого вопроса и количество вариантов
- Какой вариант ответа пробуется
- Результат каждой попытки

Без DEBUG режима (по умолчанию) видите только:
- Точки при пролистывании
- Сообщение о решении вопроса в конце

## Логирование

Все действия скрипта логируются в файл:

```
bruteforcer_YYYYMMDD_HHMMSS.log
```

Например: `bruteforcer_20260407_120000.log`

Логи содержат полную информацию о работе скрипта и помогают при отладке проблем.

## Решение проблем

### Проблема: "selenium not found"

Решение:
```
pip install --upgrade selenium webdriver-manager
```

### Проблема: Браузер не открывается

1. Проверьте, что Google Chrome установлен
2. Если Chrome в нестандартном месте, отредактируйте bruteforcer.py:
   ```python
   options.add_argument(r"--binary-location=C:\Program Files\Google\Chrome\Application\chrome.exe")
   ```

### Проблема: Кнопка не нажимается

1. Включите DEBUG режим (y)
2. Посмотрите логи в файле bruteforcer_*.log
3. Проверьте, что используется правильная ссылка на лекцию

### Проблема: Скрипт зависает

1. Проверьте интернет-соединение
2. Убедитесь, что открыта именно первая страница лекции
3. Посмотрите последние строки логи файла

## Использование собственного профиля Chrome

Если нужно использовать свой профиль Chrome (с сохраненными данными авторизации), отредактируйте bruteforcer.py:

1. Найдите строку:
   ```python
   # options.add_argument("--headless")
   ```

2. Добавьте выше неё:
   ```python
   options.add_argument(r"--user-data-dir=C:\Users\ВАШ_ЛОГИН\AppData\Local\Google\Chrome\User Data")
   options.add_argument("--profile-directory=Default")
   ```

3. Замените "ВАШ_ЛОГИН" на ваше имя пользователя Windows

## Поддерживаемые типы вопросов

Скрипт работает с вопросами типа:
- Один правильный ответ (radio buttons)

Не работает с:
- Несколько правильных ответов (checkboxes)
- Открытые вопросы (текстовое поле)
- Числовые ответы

## Как использовать сохраненные учётные данные?

1. **Первый запуск:**
   - Скрипт спросит: "Введите email/логин: "
   - Введите ваш email/логин для ВеК.ЭТУ
   - Затем спросит пароль (вводится невидимо)
   - Данные сохранятся в файл `credentials.json`

2. **Последующие запуски:**
   - Данные загружаются автоматически из `credentials.json`
   - Вам не нужно их переводить заново
   - Если вы изменили пароль, удалите файл `credentials.json` и запустите скрипт заново

3. **При редиректе на страницу логина:**
   - Скрипт автоматически вводит сохраненные данные
   - Авторизация происходит без вашего участия
